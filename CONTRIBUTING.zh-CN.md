## 项目架构概览

在贡献之前，请先理解系统：

```
3.7V 锂电池 → [防反接保护] → MT3608 升压 → 7.35V（电机轨）
                                        → LDO → 5V（逻辑轨）

5V 逻辑轨：
  NE555-1（多谐振荡，~4.8 Hz） →  OUT1 → 74HC04 → /OUT1  → L293D-1 → 电机1（前缘段）
  NE555-2（多谐振荡，~4.8 Hz） →  OUT2 → 74HC04 → /OUT2  → L293D-2 → 电机2（中段）
  NE555-3（多谐振荡，~4.8 Hz） →  OUT3 → 74HC04 → /OUT3  → L293D-3 → 电机3（后缘段）

  每个电机驱动一个曲柄摇杆四连杆机构，
  使对应翅膀段落以 ~70° 幅度、~4.8 Hz 频率往复扑动。
```

核心约束：**无MCU、无固件、无可编程逻辑**。所有控制均为分立模拟/数字电路。贡献应尊重这一设计理念。

## 开发工作流

### 分支管理

1. **Fork** 本仓库。
2. 从 `main` 创建功能分支：
   ```
   git checkout -b feature/你的功能名称
   ```
   分支命名前缀：
   - `feature/` — 新功能或设计变更
   - `fix/` — 缺陷修复（如电阻值错误、尺寸误差）
   - `docs/` — 仅文档变更
   - `media/` — 添加或更新图片/视频/渲染图

### 提交信息

遵循以下格式：
```
<领域>: <祈使语气摘要>

[可选详细正文，解释为什么，而非仅仅是什么。]
```

示例：
- `pcb: 将 MT3608 输出电容增大至 22µF`
- `mech: 为 central_support 电机安装座添加圆角`
- `docs: 修正复制指南中的引脚编号`
- `media: 添加 7.35V 电源轨的示波器截图`

### Pull Request

1. **保持 PR 聚焦。** 每个 PR 只做一个逻辑变更。
2. 在适当情况下**关联 Issue**。
3. **对于设计变更**：包含修改前后的截图、Gerber 对比或 STEP 比较渲染图。
4. **对于文档变更**：验证所有内部链接（`[文字](路径)`）均有效。
5. **等待审核。** 维护者将审核你的 PR。对于机械/PCB 变更，审核可能需要更长时间，因为需要打开 CAD/EDA 文件。

## 领域专属指南

### 电子（PCB）

**接受的设计工具：**
- EasyEDA / LCEDA（`.eprj2`） — 主要设计工具
- Altium Designer（`.schdoc`、`.pcbdoc`） — 辅助工具，用于重新捕获

**修改 PCB 时：**

1. **切勿破坏板外形** — 必须与机械框架保持兼容。板外形匹配 `central_support` 几何。
2. **原理图优先** — 在推送到 PCB 布局之前先更新原理图并重新标注。
3. **布局变更后重新导出所有输出**：
   - Gerber 文件 → `electronics/output/gerber/`
   - 贴片坐标 → `electronics/output/pick_and_place/`
   - BOM → `electronics/output/bom/`
4. **电机连接器封装** — XH-2A（P2.50 mm 间距）。未经与机械 CAD 协调（布线路径），不得更改。
5. **电源部分隔离** — 保持升压转换器环路（MT3608 + L1 + D1 + 输出电容）尽可能紧凑。不要让大电流走线穿过 NE555 时序部分下方。

**原理图变更检查清单：**
- [ ] 含有元件数值（R、C、L 及公差）
- [ ] 功率额定值已检查（尤其升压电感饱和电流）
- [ ] 时序已验证（NE555 多谐振荡：f = 1.44 / ((R_A + 2R_B) × C)）
- [ ] 已生成新 Gerber 文件
- [ ] BOM 已更新供应商型号

### 机械（CAD）

**工具：** SolidWorks 2025（`.SLDPRT`、`.SLDASM`）。如果使用不同版本的 SolidWorks，请在 PR 中注明。

**修改机械零件时：**

1. **在模型中保留参数** — 使用方程和设计表，而非硬编码尺寸。
2. **为你修改的零件导出更新后的 STEP/STL**：
   - 3D 打印零件 → `mechanical/output/3dp/`（STEP 格式）
   - 激光切割零件 → `mechanical/output/laser/`（STL 格式）
3. **配合参考** — 不得在未记录所需变更的情况下，更改装配体中已有的配合面/轴/点。
4. **紧固件兼容性** — 仅使用已指定的 M2/M3 紧固件。未经讨论不得引入新螺丝规格。

**3D 打印注意事项：**
- 0.2 mm 层高（标准）
- 0.4 mm 喷嘴（标准）
- 当前所有零件无需支撑（已为可打印性设计）
- 验证：至少在提交 PR 前打印并试装一个样品

### 文档

- **英文**为主要语言。中文（zh-CN）翻译在 `docs/zh-CN/` 和 `README.zh-CN.md` 中并行维护。
- 使用 **Markdown**（GitHub-Flavored Markdown 扩展）。
- **图片引用**：使用相对路径。将图片放入 `media/photos/` 并归入子目录（`project/`、`reference/`）。
- **图表**：框图优先使用 Mermaid（markdown 内嵌）；原理图/布局使用带标注的截图。

## 测试与验证

对于电子变更，请验证：

| 测试 | 设备 | 检查内容 |
|------|-----------|---------------|
| 电源轨 | 万用表 | 3.7 V 输入、7.35 V 升压、5.0 V LDO |
| 振荡器输出 | 示波器 | 每路 NE555 为 ~4.8 Hz、0 V 至 ~5 V 方波 |
| 相位反相 | 双通道示波器 | NE555 OUT 与 74HC04 /OUT — 互补 |
| 电机驱动 | 示波器 + 电机 | 电机两端正/反向电压 |

对于机械变更，请验证：

| 测试 | 检查内容 |
|------|---------------|
| 装配检查 | 所有零件无干涉组装 |
| 运动测试 | 连杆在完整运动范围内自由转动 |
| 可打印性 | 无 > 45° 悬垂、无孤岛、无悬浮几何 |
| 重量 | 装配体保持在目标质量范围内 |

## 报告缺陷

使用**缺陷报告** Issue 模板。

包含以下内容：
1. **你期望的结果** 与 **实际发生的现象**
2. **硬件详情**：PCB 版本、3D 打印机设置、供电电压
3. **测量数据**：示波器波形、万用表读数
4. **照片** — 你的电路板/装配体展示问题的照片
5. **复现步骤**

## 功能请求

使用**功能请求** Issue 模板。

提出建议时请考虑以下约束：
- **无MCU / 固件** — 这是核心设计理念。需要微控制器的提案可能被拒绝（但欢迎 Fork！）。
- **功率预算** — 总电流消耗受限于单节 3.7 V 锂电池和 MT3608 的能力。
- **重量预算** — 每增加一克都会减少参考值 0.16 N 的可用升力。
